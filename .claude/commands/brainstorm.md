# /brainstorm

壁打ち・ブレインストーミングの対話セッションを行う。

## 引数

- `$ARGUMENTS` - テーマ（省略可）

## 進め方

### 1. テーマ確認
- 引数があればそのテーマで開始
- なければ「何について考えたいですか？」と聞く

### 2. コンテキスト収集（任意）
- テーマに関連する過去の議論が Slack にありそうなら `slack:find-discussions` で検索
- 関連する過去のメモが `inbox/notes/` や `knowledge/decisions/` にないか確認
- 見つかった情報は「こんな過去の議論がありました」と共有し、壁打ちの材料にする
- Slack 連携が未設定の場合や、不要そうならスキップ

### 3. 対話モード
- 相槌を打ちながら深掘り質問をする
- ユーザーの考えを整理する手伝いをする
- 適度に要約を挟む

### 4. クロージング
- 「まとまってきましたね」と声かけ
- 「保存しますか？」と確認

### 5. 保存（希望時）
- `inbox/notes/YYYY-MM-DD_[テーマ].md` に保存

## 保存フォーマット

```markdown
# [テーマ] - YYYY-MM-DD

## 背景・きっかけ
- ...

## 議論のポイント
- ...

## 結論・ネクストアクション
- ...

---
captured by /brainstorm
```

## 深掘り質問の例

- 「それはなぜですか？」
- 「具体的にはどういうことですか？」
- 「他に選択肢はありますか？」
- 「一番大事なのはどれですか？」
- 「誰に影響がありますか？」
- 「期限はありますか？」

## ルール

- ユーザーの発言を否定しない
- 答えを出すのではなく、整理を手伝う
- 長くなりすぎたら途中で要約を挟む
