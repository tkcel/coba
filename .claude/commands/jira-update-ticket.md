# /jira-update-ticket

Jira チケットのステータス変更・コメント追加を行う。必ずユーザー確認後に実行する。

## 引数

- `$ARGUMENTS` にチケットキーと操作内容を指定。

## 利用可能チェック

実行前に `./scripts/jira.sh myself` で接続確認する。
失敗した場合はセットアップ手順を案内して終了。

## 対応パターン

### ステータス変更

| トリガー | 対応 |
|----------|------|
| `/jira-update-ticket PROJ-123 完了` | Done に変更 |
| `/jira-update-ticket PROJ-123 進行中` | In Progress に変更 |
| `/jira-update-ticket PROJ-123 ステータス変更` | 遷移先を選択して変更 |

**フロー:**

1. `./scripts/jira.sh transitions PROJ-123` で利用可能な遷移先を取得
2. ユーザーが指定したステータスに対応する遷移IDを特定:
   - 「完了」「done」→ "Done" を含む遷移
   - 「進行中」「着手」→ "In Progress" を含む遷移
   - 「レビュー」→ "Review" を含む遷移
   - 不明な場合 → 一覧を表示して選択
3. ユーザーに確認:
   ```
   PROJ-123「○○の実装」を In Progress → Done に変更します。よろしいですか？
   ```
4. 確認後に実行:
   ```bash
   ./scripts/jira.sh transition PROJ-123 <transitionId>
   ```
5. 結果を報告:
   ```
   PROJ-123 のステータスを Done に変更しました！
   ```

### コメント追加

| トリガー | 対応 |
|----------|------|
| `/jira-update-ticket PROJ-123 コメント ○○` | コメントを追加 |
| `/jira-update-ticket PROJ-123 に「○○」とコメント` | コメントを追加 |

**フロー:**

1. コメント内容を確認:
   ```
   PROJ-123 に以下のコメントを追加します。よろしいですか？

   > ○○の件、確認しました。問題ありません。
   ```
2. 確認後に実行:
   ```bash
   ./scripts/jira.sh comment PROJ-123 "○○の件、確認しました。問題ありません。"
   ```
3. 結果を報告

### /evening からの一括提案

`/evening` → `/jira-check evening` の結果を受けて、ステータス変更を提案される場合がある。

**フロー:**
1. `/jira-check evening` が変更候補を提示:
   ```
   今日の定例で話した PROJ-123、Done にする？
   ```
2. ユーザーが承認したら本スキルでステータス変更を実行
3. 拒否したらスキップ

### ダッシュボード更新

Jira チケットの更新後、**必ず** 当日の `daily/YYYY-MM-DD.md` の TODO セクションに変更を反映する。

- ステータス変更 → チケットを該当するステータスセクションに移動（例: 未着手 → 進行中、進行中 → 完了）
- 完了の場合 → `✅ 完了` セクションに移動し、完了日を記載
- ファイルが存在しない場合 → `/generate-daily` の実行を提案

## ルール

- **ステータス変更・コメント追加は必ずユーザーに確認してから実行**（確認なしで実行しない）
- 実行前に `./scripts/jira.sh issue <KEY>` でチケットの現在の状態を確認し、変更前後を明示する
- `.claude/rules/jira.md` のルールに従う
- `scripts/jira.sh` が失敗した場合はエラーメッセージを表示
- 秘書から呼ばれた場合は秘書の口調で対話

## 呼び出し元

| スキル | 状況 |
|--------|------|
| `/secretary` | 「PROJ-123を完了にして」「コメントして」 |
| `/evening` | Jira チケット確認後のステータス変更提案 |
