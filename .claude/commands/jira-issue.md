# /jira-issue

Jira チケットの詳細表示・検索・キー補完を行う。閲覧専用スキル。

## 引数

- `$ARGUMENTS` にチケットキー、検索キーワード、または JQL を指定。

## 利用可能チェック

実行前に `./scripts/jira.sh myself` で接続確認する。
失敗した場合は「Jira: 取得できませんでした」と報告してスキップ。

## 対応パターン

### チケット詳細表示

| トリガー | 対応 |
|----------|------|
| `/jira-issue PROJ-123` | チケットの詳細を表示 |

**実行コマンド:** `./scripts/jira.sh issue PROJ-123`

**出力フォーマット:**

```markdown
## PROJ-123: ○○の実装

| 項目 | 値 |
|------|-----|
| タイプ | Task |
| ステータス | In Progress |
| 優先度 | High |
| 担当者 | 山田 太郎 |
| 報告者 | 山田 太郎 |
| 作成日 | 2026-03-01 |
| 更新日 | 2026-03-11 |
| 期限 | 2026-03-15 |
| ラベル | backend, urgent |

### コメント（直近3件）
- **山田** (03/10): 確認しました。問題ありません。
- **山田** (03/08): 対応中です。
```

### キーワード検索

| トリガー | 対応 |
|----------|------|
| `/jira-issue 検索 ○○` | キーワードでチケット検索 |
| `/jira-issue search ○○` | キーワードでチケット検索 |

**実行コマンド:**
```bash
./scripts/jira.sh search "text ~ \"○○\" ORDER BY updated DESC"
```

**出力:** チケット一覧をテーブル形式で表示。

### JQL 検索

| トリガー | 対応 |
|----------|------|
| `/jira-issue JQL <JQL文>` | JQL を直接実行 |

**実行コマンド:** `./scripts/jira.sh search "<JQL>"`

### プロジェクト一覧

| トリガー | 対応 |
|----------|------|
| `/jira-issue プロジェクト` | プロジェクト一覧を表示 |
| `/jira-issue projects` | プロジェクト一覧を表示 |

**実行コマンド:** `./scripts/jira.sh projects`

### ユーザー検索

| トリガー | 対応 |
|----------|------|
| `/jira-issue ユーザー ○○` | ユーザーを検索 |
| `/jira-issue users ○○` | ユーザーを検索 |

**実行コマンド:** `./scripts/jira.sh users "○○"`

### 議事録中のチケットキー補完

`/format-meeting` から呼ばれた場合、議事録中のチケットキー（`[A-Z]+-\d+` パターン）に対してタイトルを補完する。

**フロー:**
1. 議事録テキストからチケットキーを抽出（正規表現: `[A-Z]+-\d+`）
2. 各キーに対して `./scripts/jira.sh issue <KEY>` でタイトルを取得
3. `PROJ-123` → `PROJ-123: ○○の実装` の形式で補完
4. 取得できなかったキーはそのまま残す

## ルール

- **このスキルは閲覧専用**。チケットの変更・作成は行わない
- `.claude/rules/jira.md` のルールに従う
- `scripts/jira.sh` が失敗した場合はスキップ（フロー全体は止めない）
- 秘書から呼ばれた場合は秘書の口調で報告

## 呼び出し元

| スキル | 状況 |
|--------|------|
| `/secretary` | 「PROJ-123 見せて」「チケットの詳細」「Jiraで○○を検索」 |
| `/format-meeting` | 議事録中のチケットキーのタイトル補完 |
| `/jira-create-ticket` | プロジェクト一覧の確認 |
