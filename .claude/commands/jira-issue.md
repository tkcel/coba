# /jira-issue

Jira チケットの詳細を表示する。議事録のチケットキー補完にも使う。

## 引数

- `$ARGUMENTS` - チケットキー（例: `PROJ-123`）。複数指定可（スペース区切り）

## 前提確認

1. `scripts/jira.sh` が存在するか確認
2. `~/.config/jira/credentials` が存在するか確認
3. いずれかがなければ「Jira: 未設定です。`/onboarding` で設定できます」と返して終了

## チケット詳細表示

```bash
./scripts/jira.sh issue PROJ-123
```

**出力フォーマット:**

```markdown
### PROJ-123: ○○の実装

| 項目 | 値 |
|------|-----|
| プロジェクト | PROJ |
| タイプ | Task |
| ステータス | In Progress |
| 優先度 | High |
| 担当 | 根本 貴志 |
| 報告者 | 山田 太郎 |
| 期限 | 2026-03-15 |
| 作成日 | 2026-03-01 |
| 更新日 | 2026-03-11 |

**説明:**
○○の実装を行う

**直近のコメント:**
- 山田さん (03/10): レビューお願いします
- 根本さん (03/11): 確認しました
```

## 複数チケット指定

スペース区切りで複数指定された場合、各チケットを順番に表示。

```
/jira-issue PROJ-123 PROJ-456
```

## `/format-meeting` からの呼び出し（キー補完）

議事録中の Jira チケットキー（`PROJ-123` 形式の文字列）を検出した場合に呼ばれる。

**動作:**
1. チケットキーごとに `./scripts/jira.sh issue <KEY>` でタイトルを取得
2. `PROJ-123` → `PROJ-123: ○○の実装` の形式に補完して返す
3. 取得失敗時はキーのまま返す（補完しない）

**例:**

```
入力: 「PROJ-123 について議論した」
出力: 「PROJ-123: ○○の実装 について議論した」
```

## エラー時の動作

- チケットが見つからない (exit code 2): 「PROJ-123: チケットが見つかりません」
- API エラー: エラー内容を表示
- キー補完モードでのエラー: キーのまま返す（呼び出し元の処理を止めない）

## ルール

- `.claude/rules/jira.md` のフォーマットに従う
- 詳細表示時はコメントは直近3件まで
