# /format-meeting

生の会議メモを構造化された議事録フォーマットに整形する。

## 対象

- `$ARGUMENTS` で指定されたファイル
- 指定なしの場合は `inbox/meetings/` 内の未整形ファイル

## 出力フォーマット

```markdown
# [会議名] YYYY-MM-DD

- 日時: YYYY-MM-DD HH:MM〜HH:MM
- 参加者: （不明なら「要確認」）
- 種別: 定例MTG / 1on1 / 開発系MTG / 外部MTG / その他

## アジェンダ
1. ...

## 議論の要約
### [トピック名]
- ...

## 決定事項
- ○○を実施する（担当: XX, 期限: MM/DD）

## TODO
- [ ] ○○（担当: XX, 期限: MM/DD）

## 備考
- ...
```

## 補完ルール

- `knowledge/people.md` を参照して人名を正式名称に統一
- `knowledge/glossary.md` を参照して略称を展開
- Google Calendar の予定情報が使えるなら、会議名・日時・参加者を補完
- 議事録中にJiraチケットキー（例: PROJ-123）が言及されていたら、`/jira-issue` でタイトルを取得して補完（Jira が利用できない場合はスキップ）

## 注意

- 情報が足りない項目は「要確認」と記載。**推測で埋めない**
- 整形後は元のファイルを上書き
- 走り書きレベルのメモでも、読み取れる範囲で最善を尽くす
