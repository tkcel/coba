# /optimize-rules

蓄積されたデータを分析し、ルールの最適化を提案・実行する。

## 引数

- `$ARGUMENTS` - 対象ルール（省略時は全ルール）。例: `daily`, `meetings`, `tmp`

## 分析対象

| ルール | 分析ソース | 最適化内容 |
|--------|-----------|-----------|
| `daily.md` | Google Calendar、daily/ | MTG分類ルールの追加・調整 |
| `weekly.md` | weekly/、inbox/meetings/ | MTGダイジェストのカテゴリ調整 |
| `meetings.md` | inbox/meetings/ | 議事録フォーマットの改善 |
| `tmp.md` | inbox/ の振り分け結果 | 振り分けルールの精度向上 |
| `knowledge.md` | knowledge/ | ナレッジ構造の改善提案 |

## 実行フロー

```
1. 対象ルールの現在の設定を読み込み
      ↓
2. 関連データを収集・分析
      ↓
3. パターンを検出
      ↓
4. 最適化案を提示
      ↓
5. ユーザー承認後、ルールを更新
```

## 分析例

### daily.md（MTG分類ルール）

**分析:**
```
Google Calendar の過去30日を分析:
- 「レビュー」を含む予定: 8件 → 未分類
- 「ふりかえり」を含む予定: 4件 → 未分類
- 「Sprint」を含む予定: 6件 → 未分類
```

**提案:**
```markdown
以下のルールを追加しますか？

| パターン | 種別 |
|----------|------|
| 「レビュー」「Review」 | レビューMTG |
| 「ふりかえり」「振り返り」「Retro」 | ふりかえり |
| 「Sprint」「スプリント」 | スプリント系 |
```

### weekly.md（MTGダイジェスト）

**分析:**
```
weekly/ の過去4週分を分析:
- 「チーム定例」が毎週出現 → 定例系に追加済み ✓
- 「デザインレビュー」が週2回出現 → 未カテゴリ
- 「All Hands」が月1回出現 → 未カテゴリ
```

**提案:**
```markdown
MTGダイジェストに以下のカテゴリを追加しますか？

### レビュー系（新規）
- デザインレビュー（曜日）: ...

### 全社系（新規）
- All Hands（月1）: ...
```

### tmp.md（振り分けルール）

**分析:**
```
inbox/other/ に振り分けられたファイルを分析:
- 「見積」「見積もり」を含むファイル: 5件
- 「契約」「NDA」を含むファイル: 3件
```

**提案:**
```markdown
以下の振り分けルールを追加しますか？

| 判定 | 振り分け先 | 判定基準 |
|------|-----------|----------|
| 見積もり | inbox/tasks/ | 「見積」「Quote」 |
| 契約関連 | inbox/other/ | 「契約」「NDA」「覚書」 |
```

## 出力

ルール更新後、変更内容をサマリー:

```
## ルール最適化完了

### daily.md
- MTG分類ルールに3パターン追加

### weekly.md
- 「レビュー系」カテゴリを追加
- 「全社系」カテゴリを追加

次回の /generate-daily から反映されます。
```

## ルール

- **必ずユーザー承認を得てから更新**
- 提案は具体的なデータ（件数・パターン）と共に提示
- 既存ルールとの重複がないかチェック
- 更新後はバックアップとして変更前の内容をコメントで残す
