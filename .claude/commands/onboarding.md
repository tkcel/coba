# /onboarding

新規ユーザーがこのワークスペースを自分用にセットアップするための対話型オンボーディング。

## フロー

### 1. ウェルカムメッセージ

```
COBAへようこそ！
このワークスペースを使い始める準備をしましょう。
いくつか質問させてください。
```

### 2. Google Calendar 接続確認

- `gcal_list_calendars` を実行してカレンダー一覧を取得
- 成功したら「カレンダー接続OK」と表示
- 失敗したら接続方法を案内

### 3. 自己紹介ヒアリング（対話形式）

以下の情報を順番に聞いていく。雑談っぽく、フレンドリーに。

#### 基本情報（必須）
1. **名前**: 「まず、お名前を教えてください」
2. **メールアドレス**: 「仕事で使っているメールアドレスは？」
3. **チーム・役割**: 「どんなチームで、どんな役割をしていますか？」

#### 働き方（任意）
4. **コアタイム**: 「普段何時頃から働いてますか？集中したい時間帯は？」
5. **ルーティン**: 「在宅日やルーティンはありますか？」

#### 秘書の設定（任意）
6. **秘書の名前**: 「秘書に名前をつけてあげてください（例: アイ、さくら、マイク など）」
   - 未設定の場合は「秘書」として振る舞う

#### AIへの指示（任意）
7. **話し方の好み**: 「AIにどんな話し方をしてほしいですか？（例: カジュアル、簡潔に、など）」
8. **判断の委任**: 「どこまで判断を任せてOKですか？」

### 4. me.md に書き込み

収集した情報を `knowledge/me.md` に書き込む。

```markdown
# 自分について

このワークスペースのオーナー情報。AIがパーソナライズするために使う。

## 基本情報

| 項目 | 値 |
|------|-----|
| 名前 | 山田 太郎 |
| メール | taro@example.com |
| 役割 | 開発チーム・エンジニア |

## 働き方

- コアタイム: 10:00-19:00
- 集中時間: 午前中
- 在宅: 水・金

## 秘書

| 項目 | 値 |
|------|-----|
| 名前 | アイ |

## AIへの指示

- カジュアルに話してOK
- 軽微な修正は確認なしで進めてよい
- 重要な判断は必ず確認
```

### 5. people.md にも追記

自分の情報を `knowledge/people.md` にも追記する（チームメンバーとして参照できるように）。

### 6. 完了メッセージ

```
セットアップ完了！

これで以下のコマンドが使えます：
- /generate-daily ... 今日のアジェンダを生成
- /triage-tmp ... tmp/ のファイルを整理
- /wrap-evening ... 1日の振り返りを生成

まずは /generate-daily を試してみてください。
```

## ルール

- 対話形式で進める（一度に全部聞かない）
- 必須項目: 名前、メールアドレス、チーム・役割
- 任意項目: 働き方、AIへの指示（後から埋めてもOK）
- 任意項目は「後で設定する」を選べるようにする
- 既に me.md がある場合は上書きするか確認
