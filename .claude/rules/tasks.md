# タスク管理ルール

## 2つのソース、1つのダッシュボード

タスクには2つのソースがあり、`daily/` の TODO セクションで **統合ビュー** として表示する。

| ソース | 管理場所 | 用途 |
|--------|----------|------|
| Jira | Jira Cloud（API経由） | 開発チケット、プロジェクトタスク |
| ローカル | `inbox/tasks/current.md` | Jira に乗らない雑務・依頼・メモ的タスク |

**原則:**
- Jira のタスクを `current.md` にコピーしない（二重管理禁止）
- ローカルタスクを Jira に自動起票しない（ユーザーが `/jira-create-ticket` で明示的に行う）
- ダッシュボード（daily/ の TODO）が唯一の統合ビュー

## ファイル構成

```
inbox/tasks/
├── current.md     ← ローカルタスクのマスター
└── archive/       ← 過去の日付別ファイル（参照用）
```

## current.md のフォーマット

```markdown
# タスク一覧

| タスク | 担当 | ステータス | 追加日 | 期限 | 完了日 | 備考 |
|--------|------|----------|--------|------|--------|------|
| ○○の対応 | 自分 | 未着手 | MM/DD | MM/DD | - | - |
| △△の確認 | 自分 | 進行中 | MM/DD | MM/DD | - | Notionリンクなど |
| ××の件 | 自分 | 完了 | MM/DD | MM/DD | MM/DD | - |
| □□の検討 | 自分 | 保留 | MM/DD | 要確認 | - | 理由メモなど |
```

## ステータス

| ステータス | 絵文字 | 意味 |
|----------|-------|------|
| 未着手 | 🔲 | まだ手を付けていない |
| 進行中 | 📝 | 作業中 |
| レビュー中 | 👀 | レビュー待ち・確認待ち |
| 完了 | ✅ | 終わった |
| 保留 | ⌛️ | 一時停止中（理由があれば備考に） |

### Jira ステータスのマッピング

| Jira ステータス | ローカルステータス |
|----------------|-----------------|
| 未着手 / To Do | 未着手 🔲 |
| 進行中 / In Progress | 進行中 📝 |
| レビュー中 / In Review | レビュー中 👀 |
| 完了 / Done | 完了 ✅ |
| 保留中 | 保留 ⌛️ |

## 統合 TODO ダッシュボード（daily/ への表示）

Jira + ローカルを混ぜてステータス別にグループ化。`ソース` 列で識別する。

```markdown
## TODO

### 🔲 未着手
| ソース | タスク | 期限 | 備考 |
|--------|--------|------|------|
| DEV-64 | バックエンド用のサブドメの作成 | なし | - |
| ローカル | ○○の対応 | 要確認 | - |

### 📝 進行中
| ソース | タスク | 期限 | 備考 |
|--------|--------|------|------|
| DEV-61 | 受付管理アプリの修正 | 03/13 | - |

### 👀 レビュー中
| ソース | タスク | 期限 | 備考 |
|--------|--------|------|------|
| DEV-38 | 常駐AIエージェント用MacMiniを購入 | なし | - |

### ✅ 完了
| ソース | タスク | 完了日 | 備考 |
|--------|--------|--------|------|
| DEV-63 | デザイン室のjiraスペースを準備 | 03/11 | - |
| ローカル | ○○さんに支払い対応依頼 | 03/11 | - |

### ⌛️ 保留
| ソース | タスク | 備考 |
|--------|--------|------|
| DEV-50 | GCPプロジェクト確認 | - |
```

### ダッシュボード表示ルール

- Jira チケットの `ソース` 列にはチケットキーを Jira へのリンク付きで表示: `[DEV-64](https://<JIRA_DOMAIN>/browse/DEV-64)`（ドメインは `knowledge/me.md` の Jira セクションから取得）
- ローカルタスクの `ソース` 列には `ローカル` と表示
- Jira は `/jira-check` で API 取得。失敗時はローカルタスクのみ表示
- **自分（`knowledge/me.md` のオーナー）が担当のタスクのみ表示**
- ✅ 完了セクションには**当日完了したタスクのみ**表示（Jira: 今日 Done になったもの、ローカル: 完了日が当日）
- ⚠️ 期限超過のチケットは期限の前に `⚠️` を付ける
- Jira → ローカルの順で表示（各ステータスグループ内）

## タスクの流れ

```
Jira Cloud ──jira-check──→ ダッシュボード（daily/ TODO）
                                    ↑
current.md（ローカル）──generate-daily──┘
手動追加 → current.md ──────────────────┘
議事録 TODO ──extract-todo──→ current.md ─┘
```

## ルール

- 担当者・期限が不明な場合は「要確認」と記載
- `knowledge/people.md` を参照して担当者名を統一
- 議事録の TODO は `/extract-todo` で `current.md` に追加する
- 完了タスクが溜まってきたら `archive/` に移動して `current.md` を軽く保つ
- 秘書がタスク追加・完了・ステータス変更時に `current.md` と `daily/` の両方を更新する
- Jira チケットのステータス変更は `/jira-update-ticket` 経由で行う
