# タスク管理ルール

`inbox/tasks/current.md` がタスクの**唯一の正（マスター）**。
この1ファイルに全タスク（持ち越し含む）を集約する。
デイリーの TODO セクションはここから生成する読み取り専用ダッシュボード。

## ファイル構成

```
inbox/tasks/
├── current.md     ← 全タスクの一覧（これだけ読めばOK）
└── archive/       ← 過去の日付別ファイル（参照用）
```

## フォーマット

```markdown
# タスク一覧

| タスク | 担当 | ステータス | 追加日 | 期限 | 完了日 | 備考 |
|--------|------|----------|--------|------|--------|------|
| ○○の対応 | 根本 | 未着手 | MM/DD | MM/DD | - | - |
| △△の確認 | 根本 | 進行中 | MM/DD | MM/DD | - | Notionリンクなど |
| ××の件 | 根本 | 完了 | MM/DD | MM/DD | MM/DD | - |
| □□の検討 | 根本 | 保留 | MM/DD | 要確認 | - | 理由メモなど |
```

## ステータス

| ステータス | 絵文字 | 意味 |
|----------|-------|------|
| 未着手 | 🔲 | まだ手を付けていない |
| 進行中 | 📝 | 作業中 |
| 完了 | ✅ | 終わった |
| 保留 | ⌛️ | 一時停止中（理由があれば備考に） |

## ルール

- 担当者・期限が不明な場合は「要確認」と記載
- `knowledge/people.md` を参照して担当者名を統一
- 議事録の TODO は `/extract-todo` で `current.md` に追加する。以降 `current.md` 側が正
- 完了タスクが溜まってきたら `archive/` に移動して `current.md` を軽く保つ

## タスクの流れ

```
議事録の TODO ──extract-todo──→ current.md（マスター）──generate-daily──→ daily のダッシュボード
手動追加 ─────────────────────→ current.md（マスター）──generate-daily──→ daily のダッシュボード
```

## ダッシュボードへの反映

- デイリーの TODO セクションはステータス別にグループ化して表示（🔲 未着手 → 📝 進行中 → ✅ 完了 → ⌛️ 保留）
- ✅ 完了セクションには**当日完了したタスクのみ**表示（完了日で判定）
- タスク完了時は `current.md` の完了日を当日の日付で記入する
- 自分（`knowledge/me.md` のオーナー）が担当のタスクのみ表示
- 秘書がタスク追加・完了・ステータス変更時に `current.md` と `daily/` の両方を更新する
