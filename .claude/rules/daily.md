## MTG 分類ルール

### デフォルトルール

| パターン | 種別 |
|----------|------|
| outOfOffice | 作業ブロック |
| 参加者1名 | 個人ワーク |
| それ以外 | その他MTG |

### カスタムルール

<!-- /onboarding または /optimize-rules で追加 -->
| パターン | 種別 |
|----------|------|
| 「朝礼」「全社朝礼」 | 朝礼 |
| 「開発定例」 | 開発定例 |
| 「定例」「【定例】」「[定例]」 | 定例MTG |
| 「よもやま」「1on1」 | 1on1 |
| 「絶対作業時間」「ブロック」 | ブロック（除外） |

## ルール

- eventType が `outOfOffice` → 「作業ブロック」セクションに分離
- description にURLがあれば 📎 付きで記載
- 既に同日のファイルがある場合は上書きせず確認する
- TODO セクションは **Jira + ローカル（`inbox/tasks/current.md`）の統合ビュー** として先頭に配置（`.claude/rules/tasks.md` 参照）
- Jira チケットは `/jira-check` で API 取得し、ローカルタスクと混ぜてステータス別に表示
- **自分（`knowledge/me.md` のオーナー）が担当のタスクのみ表示**。他の人のタスクは含めない
- ✅ 完了セクションには当日完了分のみ（Jira: 今日 Done、ローカル: 完了日が当日）
- ローカルタスクの完了・追加は `inbox/tasks/current.md` で行う。Jira は `/jira-update-ticket` で変更
- `## Jira` 独立セクションは不要（TODO に統合済み）
- Jira 取得失敗時はローカルタスクのみ表示し「Jira: 取得できませんでした」と注記

## Slack まとめルール

- `## Slack まとめ` セクションはダッシュボードの末尾に配置
- Slack MCP（`slack_search_public_and_private`）で当日の自分の送受信メッセージを取得
- BOTメッセージ（Slackbot, バクラク等）は除外し、人とのやりとりのみ記載
- 「やりとり」サブセクション: チャンネル/DM別に要約（1行ずつ簡潔に）
- 「フォローが必要」サブセクション: 未返信・確認待ちのスレッドを明示
- ダッシュボード表示・更新時に最新状態に更新する（追記ではなく既存セクションを置換）
- Slack MCP が利用できない場合は「Slack まとめ: 取得できませんでした」と記載してスキップ
